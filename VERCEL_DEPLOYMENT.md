# Vercel Deployment Guide — Mahdi Dhaker Official

## Pre-Deployment Checklist

- [ ] GitHub repository connected
- [ ] `.env.local` NOT committed (in `.gitignore`)
- [ ] `.env.example` in root with placeholder values
- [ ] Build succeeds locally: `npm run build`
- [ ] No console errors in dev: `npm run dev`

---

## Step 1: Deploy to Vercel

### Option A: Via Vercel CLI
```bash
npm i -g vercel
vercel
```
Follow prompts to connect GitHub and deploy.

### Option B: Via Vercel Dashboard
1. Go to [vercel.com/new](https://vercel.com/new)
2. Click "Import Git Repository"
3. Select this repo
4. Click "Import"
5. Vercel will auto-detect Next.js and configure

---

## Step 2: Add Environment Variables to Vercel

**⚠️ CRITICAL: Without these, ticketing will NOT work in production**

### In Vercel Dashboard:

1. Go to your project → **Settings** → **Environment Variables**
2. Add these variables for **Production** environment:

| Variable | Value | Example |
|----------|-------|---------|
| `NEXT_PUBLIC_BASE_URL` | Your production domain | `https://mahdidhaker.com` |
| `KONNECT_API_KEY` | API key from Konnect dashboard | `69bfe594cd2bd...` |
| `KONNECT_WALLET_ID` | Wallet ID from Konnect dashboard | `69bfe594cd2bd...` |
| `KONNECT_API_URL` | Konnect API endpoint | `https://api.konnect.network/api/v2/payments/init-payment` |

### How to get Konnect credentials:

1. **Create merchant account** at [konnect.network/dashboard](https://konnect.network/dashboard)
2. **Wait for verification** (2–5 business days)
3. **Copy API Key** from Dashboard → Settings → API Keys
4. **Copy Wallet ID** from Dashboard → Settings → Wallet

---

## Step 3: Configure Production Environment

### Update NEXT_PUBLIC_BASE_URL

After your domain is set (e.g., `mahdidhaker.vercel.app` or custom domain), update in Vercel:

```
NEXT_PUBLIC_BASE_URL = https://mahdidhaker.vercel.app
```

This is used for payment redirects and webhooks to work correctly.

---

## Step 4: Test Ticketing Flow

1. **Deploy** → Vercel will rebuild with env vars
2. **Visit production site** → `/tickets`
3. **Add tickets** → Click "Proceed to checkout"
4. **Fill form** → Click "Pay [amount] TND"
5. **Verify redirect** → Should go to Konnect payment page

#### If stuck on checkout page:
- Check Vercel Build Logs for errors
- Verify env variables are set in Vercel Dashboard
- Confirm API keys are correct

---

## Step 5: Enable Custom Domain (Optional)

1. **In Vercel Dashboard** → Project → Settings → Domains
2. **Add your domain** → `mahdidhaker.com`
3. **Follow DNS instructions** for your registrar
4. **Wait for SSL** (auto-generated by Vercel)

After domain connects, update `NEXT_PUBLIC_BASE_URL`:
```
NEXT_PUBLIC_BASE_URL = https://mahdidhaker.com
```

## Troubleshooting

### "Module not found" error
- Vercel build failed
- Check **Build Logs** in Vercel Dashboard
- Run `npm run build` locally to debug

### Tickets page loads but payment button doesn't work
- Env variables not set
- Check **Settings → Environment Variables** in Vercel
- Verify `KONNECT_API_KEY` and `KONNECT_WALLET_ID` are present

### Konnect payment fails
- Wrong API endpoint (sandbox vs production)
- Check Konnect dashboard for correct URL
- If using sandbox: `https://api.sandbox.konnect.network/...`
- If production: `https://api.konnect.network/...`

### Payment redirects to wrong URL
- `NEXT_PUBLIC_BASE_URL` mismatch
- Make sure it matches your deployed domain exactly

## Post-Deployment

### Monitor Tickets
- Konnect Dashboard → Transactions
- Vercel Dashboard → Functions (for API logs)
- Check email for payment confirmations

### Update Content
- Edit files → Push to GitHub
- Vercel auto-redeploys (takes ~1 min)

### Rollback
- Vercel Dashboard → Deployments → Select previous → Promote

## Security Notes

✅ **Environment variables are never exposed to client**  
✅ **API keys only used server-side**  
✅ **NEXT_PUBLIC_* variables are visible in browser (but safe — only base URL)**  
✅ **All KONNECT_* variables are hidden from browser**
